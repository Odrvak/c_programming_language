# Contributing Guidelines

## How to Contribute

Contributions are welcome! Please follow these guidelines:

### Code Standards

1. **Code Style**
   - Use consistent indentation (4 spaces)
   - Follow K&R or Allman brace style consistently
   - Use meaningful variable names
   - Keep functions small and focused

2. **No Comments Policy**
   - Code should be self-documenting
   - Use descriptive function and variable names
   - Structure code logically

3. **File Naming**
   - Use lowercase with underscores: `01_example_name.c`
   - Number files sequentially within directories
   - Keep names descriptive but concise

4. **Compilation**
   - All code must compile with: `gcc -Wall -Wextra -std=c11`
   - No warnings allowed
   - Test on multiple platforms when possible

### Adding New Examples

1. Choose the appropriate category directory
2. Create a new file with the next sequential number
3. Ensure code demonstrates a single concept clearly
4. Test compilation and execution
5. Update README.md if adding a new category

### Pull Request Process

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Test thoroughly
5. Submit a pull request with a clear description

### Code Review Criteria

- Correctness
- Clarity
- Efficiency
- Portability
- Security considerations

## Questions?

Open an issue for discussion before making major changes.
